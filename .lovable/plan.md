# План: большой батч улучшений

Делаю по порядку, мелкие задачи группирую в один проход. Внутри каждой задачи — что появится у пользователя и где править в админке.

## 1. Расписание плановых курсов
- Расширяю таблицу `course_schedules`: `city` (Красноярск / Томск / Кемерово / Онлайн), `format` (очно/дистанционно), `is_default` (флаг шаблона для дистанта).
- В админке `/admin/schedules` — фильтр по городу, кнопка "Шаблон дистанта" (одна запись на курс, применяется автоматически).
- На странице курса — блок "Ближайшие даты" с табами городов.

## 2. Преподавательский состав
- Новая таблица `teachers` (имя, должность, фото, регалии, био, порядок).
- Парсю состав с cpr-partner.ru/about (раздел "Наши преподаватели"), заливаю.
- Админка `/admin/teachers` — CRUD + загрузка фото.
- Публичный блок на /about и опционально привязка к курсу.

## 3. Документы организации (лицензии, благодарности)
- Таблица `org_documents` (название, тип: лицензия/благодарность/устав, файл/изображение, порядок).
- Парсю с /about/#docs и /about/#thanks.
- Админка `/admin/documents` — загрузка PDF/JPG.
- Блок на /sveden и /about — сетка превью с лайтбоксом.

## 4. Галерея
- Таблица `gallery_images` (url, alt, порядок, раздел: главная/страница).
- Админка `/admin/gallery` — drag-загрузка, сортировка.
- Блок на главной (карусель внизу) + страница `/gallery`.

## 5. Города: Томск, Красноярск, Кемерово
- Кемерово: новая страница `/kemerovo`, в каталоге показываем только БПЛА (фильтр по тегу/категории `bpla`).
- В шапке — переключатель города (cookie `city`); меняет телефон/адрес и фильтрует расписание.
- Расписание курса показывает табы по доступным городам.

## 6. Версия для слабовидящих — стандартный плагин
- Подключаю Bvi (bvi.js) — стандарт РФ, тот же, что на доноре.
- Удаляю самописный AccessibilityToggle, оставляю кнопку запуска Bvi в шапке.

## 7. Поиск по сайту
- Поле в шапке (Ctrl+K). Ищет по курсам, статьям блога, направлениям.
- Реализация: серверная функция, ILIKE по title/description + ts_vector индекс.

## 8. Куки, 152-ФЗ
- Cookie-баннер (принять/настройки/только необходимые), сохранение в localStorage.
- Страница `/cookies`, обновляю `/privacy` и форму согласия.

## 9. Загрузка расписания из Excel
- В админке курса — кнопка "Загрузить расписание (.xlsx)".
- Парсинг на сервере (xlsx-пакет): колонки `Курс | Город | Формат | Дата начала | Дата конца | Цена`.
- Дистант → пишется в `is_default` запись; очно → отдельные даты по городам.
- Превью перед импортом, лог в `audit_log`.

## 10. Соцсети ВК
- Беру ссылки с донора, добавляю в `app_settings`, вывожу в Footer и FloatingActions.

## 11. Типы документов об обучении
- В `courses` добавляю `document_type` enum: `svidetelstvo` | `udostoverenie` | `diplom` (авто по `hours`: <16 → свидетельство, 16-249 → удостоверение, ≥250 → диплом; ручное переопределение в админке).
- На странице курса — плашка "По итогам выдаётся: Удостоверение о ПК" + превью образца документа.

## 12. Референс страницы курса (ncpo.ru)
- Подтягиваю структуру: hero с ценой/часами/документом → "Что получите" → программа (аккордеон) → преподаватели курса → расписание по городам → отзывы → FAQ → форма заявки.
- Переписываю `/courses/$slug` под эту раскладку, сохраняя текущий контент.

---

## Технические детали

- Миграции: `course_schedules` (доп.поля), `teachers`, `org_documents`, `gallery_images`, `cookie_consents` (опц.), `courses.document_type`.
- Storage buckets: `teachers`, `org-documents`, `gallery` (public read, admin write).
- Парсинг донора — через server fn + Firecrawl/fetch, разовый seed.
- Bvi подключаю через CDN-скрипт в `__root.tsx`.
- Excel импорт — пакет `xlsx` (SheetJS), парсинг внутри `createServerFn` с `requireSupabaseAuth` + проверкой роли admin.
- Поиск — сначала простой ILIKE, при росте контента добавлю pg_trgm.

## Порядок выполнения

Группирую в 3 захода, чтобы не ломать сайт:

1. **Бэкенд + админка**: миграции (1, 2, 3, 4, 5, 11), CRUD-страницы админки, парсинг донора (преподаватели + документы).
2. **Публичные фичи**: переключатель городов, Кемерово, галерея, ВК, типы документов, редизайн страницы курса (12).
3. **UX/прочее**: Bvi, поиск, cookie-баннер, импорт Excel (9).

Подтверди — стартую с шага 1. Если по какому-то пункту хочешь иначе (например, не Bvi а оставить мой тоггл, или Кемерово сразу с полным каталогом) — скажи, поправлю.
